    • QTI 3 scope: the 1EdTech Delivery conformance ladderADR-0002
    • Capability gate: unsupported content is reported, never silently droppedADR-0003
    • Staged response-processing interpreter, spec-strict by defaultADR-0004
    • assessmentTest semantics live here, as a headless test controllerADR-0005
    • PCI hosting is an opt-in trust boundary with an injectable module registryADR-0006
    • QTI Results Reporting: views in qti-react, the binding in qti-xmlADR-0006
    • PCI loading: install model now, hash-pinned catalog as a policy layer laterADR-0007
    • PNP/AfA and the catalog subsystem: activation in pnp, presentation in the runtimeADR-0008
    • Full-spec completion: every binding reads, every registered root normalizesADR-0009
    • ASI export serializers: every readable binding now writes, gated by a corpus round tripADR-0010
    • Official-XSD conformance hardening laneADR-0011
    • Generic PCI delivery: hash-pinned module catalog + state persistenceADR-0012
    • Spec Coverage Map: literal-schema denominator, Zod provenance, conformance catalogADR-0013
    • Docs site: Astro Starlight, monorepo source deployed to a dedicated org-pages repoADR-0014
    • Counterpart-backed interop lanes: real-system realism, not a conformance oracleADR-0015
    • Unified TypeScript release, versioning, and tooling standard (emergent / conform-ed / pgxsinkit)ADR-0016
    • Hand-authored denominators for prose-only spec familiesADR-0017
    • Caliper 1.2 — full contract modelling, Sender/Endpoint board, and an event validatorADR-0018
    • Europass / ELM v3.3 support: VC-agnostic core, all four profiles, SHACL denominator, JAdES sealADR-0019
